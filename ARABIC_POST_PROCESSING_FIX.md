# 阿拉伯语后处理 Markdown 格式保护修复

## 🐛 问题描述

之前的阿拉伯语后处理步骤会破坏 Markdown 格式，具体表现为：

1. **代码块缩进丢失**：多个空格被替换为单个空格
2. **列表格式错乱**：行首空格被移除
3. **表格对齐问题**：制表符和空格被规范化
4. **引用块格式丢失**：缩进被移除

## ✅ 修复内容

### 修改前的问题代码

```python
def _clean_arabic_text(text: str) -> str:
    # 移除所有控制字符（包括制表符 \t）
    text = re.sub(r'[\u0000-\u001F\u007F-\u009F]', '', text)
    
    # 规范化空格
    lines = text.split('\n')
    for line in lines:
        line = re.sub(r'[ \t]+', ' ', line)  # ❌ 破坏缩进
        line = line.strip()                   # ❌ 移除行首行尾空格
```

**问题**：
- 移除了制表符 `\t`（Markdown 代码块需要）
- 将多个空格替换为单个空格（破坏缩进）
- 移除行首行尾空格（破坏列表、引用等格式）

### 修改后的代码

```python
def _clean_arabic_text(text: str) -> str:
    # 移除零宽字符
    text = re.sub(r'[\u200B-\u200D\uFEFF]', '', text)
    
    # 只移除真正有问题的控制字符
    # 保留：\t (制表符), \n (换行), \r (回车)
    text = re.sub(r'[\x00-\x08\x0B-\x0C\x0E-\x1F\x7F-\x9F]', '', text)
    
    # 不再规范化空格，保留 Markdown 格式
    return text
```

**改进**：
- ✅ 保留制表符 `\t`
- ✅ 保留换行符 `\n`
- ✅ 保留回车符 `\r`
- ✅ 保留所有空格（包括多个连续空格）
- ✅ 保留行首行尾空格
- ✅ 只移除真正有问题的控制字符

## 🔍 仍然处理的问题字符

后处理仍然会移除以下有问题的字符：

### 1. 零宽字符
- `\u200B` - 零宽空格
- `\u200C` - 零宽非连接符
- `\u200D` - 零宽连接符
- `\uFEFF` - 零宽非断空格（BOM）

**原因**：这些字符不可见，但会影响文本处理和搜索

### 2. 有问题的控制字符
- `\x00-\x08` - NULL 到 BACKSPACE
- `\x0B-\x0C` - 垂直制表符、换页符
- `\x0E-\x1F` - 其他控制字符
- `\x7F-\x9F` - DEL 和扩展控制字符

**原因**：这些字符在文本中通常是错误或不需要的

## 🔧 如果还有问题

如果您发现 Markdown 格式仍然有问题，可能的原因：

### 1. MinerU 本身的识别问题
后处理不会修复 MinerU OCR 引擎本身的识别错误。

**解决方案**：
- 尝试不同的后端模式（pipeline / vlm-auto-engine / hybrid-auto-engine）
- 检查原始文档质量

### 2. 文本方向修复的影响
BiDi 转换可能会影响某些格式。

**解决方案**：
- 尝试不同的文本修复模式（auto / always / never）
- 在页面上选择"从不转换（never）"

### 3. 特殊字符问题
某些特殊的 Markdown 语法可能包含被移除的控制字符。

**解决方案**：
- 如果需要完全禁用后处理，请告诉我，我可以添加一个开关

## 📝 技术细节

### Unicode 控制字符范围

|------|------|----------|
| `\x00-\x08` | NULL 到 BACKSPACE | 移除 |

| `\x0D` (`\r`) | 回车符 | **保留** |
| `\x0E-\x1F` | 其他控制字符 | 移除 |
| `\x20-\x7E` | 可打印 ASCII | **保留** |

| `\u200B-\u200D` | 零宽字符 | 移除 |
| `\uFEFF` | BOM | 移除 |
